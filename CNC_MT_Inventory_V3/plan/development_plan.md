### **개발 계획**

**1단계: 프로젝트 설정 및 백엔드 개발 (Node.js & Express)**
1.  **`server` 폴더 설정:** Node.js 프로젝트를 초기화하고 Express.js 서버를 구성합니다.
2.  **DB 연동:** 기존 Supabase 데이터베이스와의 연동을 설정합니다.
3.  **API 엔드포인트 개발:** 재고, 부품, 입출고, 공급처, 사용자 관리를 위한 RESTful API를 설계하고 구현합니다.
4.  **인증 시스템 구축:** `bcrypt`를 이용한 비밀번호 암호화, JWT 기반의 로그인/로그아웃 API를 구현합니다.
5.  **미들웨어 구현:** API 요청에 대한 사용자 인증 및 역할 기반 접근 제어(RBAC) 미들웨어를 구현합니다.

**2단계: 프론트엔드 개발 (React & Tailwind CSS)**
1.  **`client` 폴더 설정:** Vite를 사용하여 React 프로젝트를 생성하고 Tailwind CSS를 설정합니다.
2.  **기본 컴포넌트 개발:** 버튼, 입력 폼, 모달, 데이터 테이블 등 재사용 가능한 UI 컴포넌트를 제작합니다.
3.  **라우팅 설정:** `React Router`를 사용하여 페이지 간의 이동(내비게이션)을 구현합니다.
4.  **페이지 개발:** 기획서에 명시된 각 핵심 기능(재고, 부품, 입출고 등)에 대한 페이지를 개발합니다.
5.  **API 연동:** 백엔드 API와 통신하여 데이터를 주고받는 로직을 구현합니다.
6.  **상태 관리:** `Zustand` 또는 `Redux Toolkit`을 사용하여 사용자 정보 등 전역 상태를 관리합니다.

**3단계: 통합, 테스트 및 배포**
1.  **통합 테스트:** 프론트엔드와 백엔드를 연동하여 전체 시스템의 흐름을 테스트하고 디버깅합니다.
2.  **최적화:** 성능 및 사용자 경험을 개선합니다.
3.  **배포:** Docker를 사용하여 애플리케이션을 컨테이너화하고, Vercel(프론트엔드)과 Render(백엔드) 등의 클라우드 플랫폼에 배포합니다.
